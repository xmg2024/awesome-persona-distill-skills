# 贡献指南

感谢你为本存储库做贡献。

本存储库收录的是围绕“人格蒸馏”主题的 Agent Skills，重点关注以下方向：

- 从人物、关系、对话、作品或数字痕迹中提炼表达风格、判断方式或交互模式。
- 将某种人物视角、关系经验或方法论整理成可复用的技能。
- 具有明确主题边界，适合被收录为列表项目，而不是泛泛的 AI 工具目录。

## 欢迎的贡献

- 新增符合主题的 skill 存储库链接。
- 修正失效链接、存储库名、描述或分类位置。
- 优化 README 中的表述，让分类或描述更准确。

对于“新增收录申请”，请使用 issue 表单作为唯一公开入口。维护者审核通过并添加 `approved` 标签后，工作流会自动生成 PR 供审阅。

## 不建议提交的内容

- 与“人格蒸馏”主题关系较弱的通用 AI 工具、提示词集合或资源导航。
- 缺少公开存储库、无法访问，或几乎没有可判断内容的项目。
- 明显重复、仅换壳重发，或与现有条目高度重叠但没有新增价值的项目。
- 以侵犯隐私、骚扰、冒充真实个人或规避同意为主要目的的项目。
- 纯广告、推广文案，或需要读者跳转后才能理解基本内容的提交。

## 提交前请确认

请尽量让每个条目满足下面这些标准：

- 存储库可以公开访问。
- 项目主题与本列表相关，而且放在当前分类下是合理的。
- 链接名称使用项目常用名称。
- 描述尽量控制在一句话，说明它蒸馏的对象或方法，不写宣传性措辞。
- 若项目更适合放到另一个分类，请直接按更合适的分类提交。

## 条目写法参考

如果你是在申请新增收录，请按下面的格式和风格填写 issue 表单中的汉英语描述。维护者添加 `approved` 标签后，工作流会据此自动生成 README 条目。

如果你是在直接修订现有条目，也请沿用当前 README 的列表格式：

```md
- [项目名](https://github.com/owner/repo) - 一句话说明项目蒸馏的对象、来源或用途。
```

建议保持这些写法：

- 先写“它是什么”，再写“它做什么”。
- 描述具体，不要只写“高质量”“强大”“好用”。
- 与周围条目的语言风格保持一致。

## 提交流程

1. 如果你是申请新增收录，请先提交 issue 表单，完整填写项目链接、分类、汉英语描述与收录理由。
2. 维护者审核后，如认为适合收录，会为该 issue 添加 `approved` 标签，并由工作流自动生成 PR。
3. 如果你是在修复现有条目、文档或失效链接，可直接 fork 存储库、创建分支并提交 PR。
4. 对需要直接提交 PR 的改动，请同步更新 [`README.md`](./README.md) 和 [`README.en.md`](./README.en.md) 中受影响的内容；如果改动影响收录标准或协作方式，也同步更新本文件。
5. 本地运行格式化，并检查条目写法、链接和文案风格与现有内容一致。

## 本地检查

提交前建议先运行：

```bash
npm ci
npm run format
```

然后重点确认：

- Markdown 列表格式与现有条目一致。
- 链接可公开访问。
- 名称与描述准确，不使用宣传性措辞。

## PR 建议

为了让审阅更快，请在 PR 中说明：

- 你新增或修改了什么。
- 它属于哪个分类，为什么放在这里。
- 如果是新增项目，它与“人格蒸馏”主题的关系是什么。
- 如果有相近条目，和现有条目的区别是什么。

## 沟通约定

- 新增收录请走 issue 表单；不要直接提交“新增收录”PR。
- 小改动、现有内容修正或失效链接修复可以直接提 PR。
- 如果你打算调整分类结构、重写大量描述，或对收录范围做明显扩展，建议先开 issue 说明思路。
- 讨论时请聚焦主题边界、可读性和读者价值。
